# ELS_hedging_V1 — 빌드 지시서 (노션 "모델링 관련 내부 회의" 1차 재현)

> 목적: 팀 노션 문서의 하이브리드 파이프라인을 **raw 데이터부터 처음부터 독립 구현**해, 팀원 결과(deeponet_hybrid R²≈0.73, per-stage R²)와 **비슷하게 나오는지 검증**. 이후 피드백 반영·추가실험은 별도.
>
> 컨벤션 확정: **변동성/상관 = 180일 역사(미래에셋 ELS 공정가치 관행)**, **할인·드리프트 = KR FRED 금리(콜·3M·10Y) Nelson-Siegel** (KRW 상품이므로 원화금리). Stage-2 = **XGBoost**(노션과 동일 조건으로 비교). 시장데이터는 **처음부터 fetch**(캐시 미사용).

---

## 0. 파이프라인 개요
```
data/raw 3 CSV ─▶ [1] 전처리(필터·스케줄벡터) ─▶ product_master
                        │
   yfinance/FRED ─▶ [2] 시장데이터(180일 vol·corr + KR NS곡선) ─▶ +sig/rho/u0..9/r/mom6m
                        │
                  [3] MC 이론가(worst-of 오토콜 GBM) ─▶ +MC
                        │
                  [4] recent_margin(발행전 90일 fair−MC) ─▶ +recent_margin
                        │
       [5] 데이터셋 3종: ml / deeponet / (pi_deeponet 선택)
                        │
       [6] 모델: Stage-1 DeepONet-Curve(MC 재현) + Stage-2 XGB(잔차)  + 벤치(ridge/gbm…)
                        │
       [7] walk-forward 4-fold 평가: R²·MAE·Spearman·per-stage R²
```

권장 모듈 구조:
```
els_hedging_v1/
  config.py        data_prep.py     market_data.py   mc_engine.py
  features.py      datasets.py      deeponet.py      benchmark.py
  splits.py        metrics.py       run_all.py
data/{raw, cache, out}
```

## 1. 전처리 (`data_prep.py`)
- 원본: `LAKE_V2_DART_AUTO_CALL.csv`(상품), `LAKE_V2_DART_SCHD_INFO.csv`(스케줄), `LAKE_V2_DART_UDLY_INFO.csv`(기초자산).
- **필터** (노션 기준, 목표 ≈ 23,151건):
  - `PRODUCT_TYPE=='ELS'`, worst-of(`STCK_MTHD=='ALL_MIN'`), 3-star(`UDRL_CNT==3`), `OPT_TYPE=='STEP'`, `CUR_CD=='KRW'`
  - `fair = FAIR_VALUE/ISU_PRC_DETAIL ∈ [0.70, 1.05]`, `tenor = (MAT_DT−ISU_DT)/365.25 ∈ [0.5, 5]`
  - ⚠ 만기는 `MAT−ISU` (DT_DIFF 아님). 3기초자산 전부 티커 매핑 가능한 상품만 남김(안 되면 제외).
  - 최종 건수가 23,151과 다르면 로그로 남기고 원인(티커 커버리지·CPN_YN 포함 여부)만 표시 — 정확히 맞추려 억지 필터 넣지 말 것.
- **스케줄 벡터**: `SCHD_TYPE==1` STRK_1을 SEQ 순 → `strk_0..11`(/100, 12패딩, 짧으면 마지막값 forward-fill). `BARR_1` 최소/100 = 낙인배리어 `B`. `ANL_RTRN/100`=coupon, `TENOR`.
- **파생**: `Kfirst=strk_0, K=strk_last, b_over_k=B/K, stepdown=Kfirst−K, nobs=상환평가횟수, cpn_spread=coupon−r(§2 후), STRK_1_first/last`.
- 키: `item=ITEM_CD`, `isu_ord=ISU_DT.toordinal()`(정렬키·피처아님), `issuer=ISU_ORG`.
- 산출: `data/cache/product_master.parquet`.

## 2. 시장데이터 (`market_data.py`) — 180일·KR금리
- **기초자산→yahoo 티커 매핑**: 지수 `EURO STOXX50→^STOXX50E, S&P500→^GSPC, HSCEI→^HSCE, KOSPI200→^KS200, Nikkei225→^N225, NASDAQ100→^NDX, DAX→^GDAXI, 항셍→^HSI`; 개별주 `삼성전자→005930.KS` 등, 미국주 `AAPL/NVDA/TSLA/AMD/…`. 못 찾으면 로그 후 상품 제외.
- **가격 fetch**: yfinance `auto_adjust=True` 종가. **429 대비 백오프 재시도**, 실패 티커는 **FinanceDataReader**(KR) 대체 시도. `data/cache/px_*.parquet`로 저장(재실행 재사용).
- **변동성·상관 (180일)**: 발행일 직전 **180 영업일** 로그수익률로
  - 개별 변동성 연율화 → 정렬 `sig1≤sig2≤sig3`, 쌍상관 `rho12/13/23`(정렬), 유효변동성 `sig_eff`(상관반영 등가중 바스켓), `sig_mean`, `rho`(평균), `mom6m`(126일 로그수익률).
- **KR 금리커브**: FRED `IRSTCI01KRM156N`(콜<24h), `IR3TIB01KRM156N`(3M), `IRLTLT01KRM156N`(10Y). 발행일 기준 이 3점으로 **Nelson-Siegel 적합** → tenor 노드 `[0.25,0.5,1,1.5,2,3,4,5,7,10]`에서 `u0..u9`, 상품 만기에서 `r`. (월간→일간 ffill)
- 산출: `product_master`에 위 컬럼 병합 → `product_market.parquet`.

## 3. MC 이론가 (`mc_engine.py`)
- worst-of 스텝다운 오토콜, 위험중립 3자산 GBM. **경로 10만**, 관측일 기준 시뮬(정확), **일별 KI 감시**, 현금흐름은 **각 시점 KR NS 할인계수로 할인**.
- 드리프트=`r`, 공분산=180일 σ·ρ의 Cholesky. S0=1 정규화, worst=3자산 min.
- 오토콜: 각 상환평가일 worst≥strk_k → (1+누적쿠폰)·DF 후 종료. 만기: worst≥최종행사가→쿠폰지급, KI(장중 worst<B 이력)→worst·DF 손실, 아니면 원금.
- BS 단순케이스로 엔진 검증(1자산·무KI). 산출: `+MC` → `product_mc.parquet`. (상품당 시간도 기록)

## 4. recent_margin
- `recent_margin_i = mean{ fair_j − MC_j : isu_ord_i−90 ≤ isu_ord_j < isu_ord_i }` (발행전 90일 인과평균, 이력없으면 0/전역백필). 미래참조 없음.

## 5. 데이터셋 (`datasets.py`) — 노션 3.1~3.3
- **공통 타깃/앵커**: `fair`(타깃), `MC`(앵커·피처아님), `recent_margin`(앵커항), `isu_ord`(split-key).
- **ml (tabular·Stage-2)**: BASE + REG + CAT (아래) + target.
- **deeponet**: branch=`u0..u9`(10) + `sig1..3,rho12/13/23,sig_eff`(7), trunk=`strk_0..11 + B + coupon + tenor`(15), aux=`r`.
- 피처그룹:
  - **BASE**(공통 수치): sig1/2/3, rho12/13/23, sig_mean, rho, r, B, K, Kfirst, coupon, tenor, nobs, cpn_spread, b_over_k, stepdown, mom6m, ACT_ISU_AMT, SB_RT, DV_RT, PRCP_GRTE_RT, KNCK_IN_GRC_PRD, ISU_DT_year, (SUB_END−SUB_START) + strk_0..11
  - **REG**(direct 전용): recent_margin, recent_mktvol(발행전90일 sig_mean평균), curve_level(u0..u9평균), curve_slope(u9−u0), curve_curv(2·u4−u0−u9), issue_intensity(발행전90일 발행건수)
  - **CAT**(원핫): ISU_ORG, RISK_GRADE, PRODUCT_TYPE, RDMP_TYPE, ISU_DT_month
- ⚠ Stage-2 타깃 = `resid = fair − MC − recent_margin`. Stage-2 입력엔 MC·recent_margin **미포함**(REG도 제외, BASE+CAT만).

## 6. 모델 (`deeponet.py`, `benchmark.py`)
- **Stage-1 DeepONet-Curve**: branch 1D-CNN(수익률곡선 u0..9) + vol·corr concat → MLP, trunk MLP(계약 15) → 내적 → **MC 재현**(타깃=MC). (허정규 논문 구조 차용)
- **Stage-2 XGBoost**: `resid` 학습(BASE+CAT). `margin̂ = recent_margin + resid̂`, `Final = MC + margin̂`. → `pip install xgboost` 필요.
- **하이브리드 변형**(노션 2.1): `deeponet_hybrid`(Stage1 DeepONet), `xgb_hybrid`(Stage1도 XGB), (선택) `deeponet_hybrid_s2don`.
- **Direct 벤치**(노션 2.2): fair 직접회귀 — bench_ridge, bench_gbm/xgb/lgbm/cat, deeponet_direct.

## 7. 검증 (`splits.py`, `metrics.py`) — 노션 3
- **walk-forward 4-fold**: 누적 train[0→60/70/80/90%], test 다음 10% → OOS 40%(≈9,261). 항상 과거→미래, look-ahead 없음.
- train 내 전처리 통계(스케일러·원핫)만 train으로 fit. (원 노션은 시간감쇠 가중 0.5^(dt/365) 사용 — 넣어도 됨.)
- 지표: **R², MAE, RMSE, Spearman ρ**, 그리고 **per-stage R²**: Stage-1(MC 재현), Stage-2(resid), Final(fair).

## 8. 재현 성공 기준 (팀원 목표치)
- Stage-1 MC 재현 R² ≈ **0.92~0.99** (DeepONet), xgb는 ~0.83~0.90
- Stage-2 resid R² ≈ **0.49**
- Final(fair) **deeponet_hybrid R² ≈ 0.72~0.74**, Spearman ρ ≈ 0.85~0.87
- Direct: ridge ~0.56, tree ~0.26~0.33, deeponet_direct 음수
→ 이 대역에 들면 "재현 성공". 크게 벗어나면 어느 단계(필터 건수·MC·피처)에서 갈렸는지 진단.

## 부록 — 재현 순서
```bash
pip install numpy pandas pyarrow matplotlib torch yfinance pandas-datareader scipy xgboost tqdm finance-datareader
python -m els_hedging_v1.data_prep
python -m els_hedging_v1.market_data
python -m els_hedging_v1.mc_engine --n-paths 100000 --jobs <코어수>
python -m els_hedging_v1.datasets
python -m els_hedging_v1.deeponet        # Stage-1
python -m els_hedging_v1.benchmark       # Stage-2 XGB + direct 벤치 + 하이브리드 조립
python -m els_hedging_v1.metrics         # per-stage R² 표 + 팀원 목표치 비교
```
> 환경변수: 실행 전 `KMP_DUPLICATE_LIB_OK=TRUE`, `PYTHONIOENCODING=utf-8` (Windows). MC는 무거우니 `--jobs`에 코어수, 반복 실험 땐 경로수↓로 빠르게.
