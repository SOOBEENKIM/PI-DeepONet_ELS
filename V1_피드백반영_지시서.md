# ELS_hedging_V1 — 피드백 반영 재정비 지시서 (이론가 중심)

> 재현 단계는 끝. 이제 **교수님 피드백을 반영한 "올바른 모델"**로 정비하고, 팀원 궤적(속도·fair−MC·per-stage R²·저가 케이스)까지 산출한다.
>
> **전제 (중요):**
> - **MC 재실행 금지.** 기존 `data/cache/product_mc.parquet` 그대로 사용. (컨벤션 180일 vol·상관 + KR FRED NS금리는 팀 현행과 동일 — 바꾸지 말 것.)
> - 먼저 **버그 수정**: DeepONet `train_fold`에서 검증 개선이 한 번도 없던 fold일 때 `best_state`가 None → `load_state_dict(None)`로 크래시(`TypeError: Expected state_dict ... NoneType`). **best_state None이면 현재 모델 상태를 그대로 쓰도록 가드**(또는 첫 epoch state를 항상 저장).

## 0. 헤드라인 원칙 (피드백)
- **우리 본질 = 이론가(Stage-1).** Stage-2/Final은 부차 → **Stage-1 DeepONet의 MC 재현을 핵심 지표로.**
- **R²만 보지 말 것 → %error(현업 관점).** "어이없는 값 안 나온다"를 보이는 framework.
- 스토리: *FV엔 마진이 붙어 pricing이 잔 안 됨 → 이론가에 대한 surrogate는 잘 됨 → 이론가 중심으로 pricing.* + 속도.

## 1. 수정 항목
1. **버그 가드**(위 §전제) 먼저.
2. **되돌리기**: 직전 `benchmark.py` 변경(RobustScaler·median fillna·time-decay 등)이 stage1_xgb 0.896→0.835, xgb_hybrid 0.338→0.160으로 **악화** → **DeepONet 안정화(타깃 표준화·grad clip·epochs200/patience20·LR감쇠)만 남기고 XGB 악화 변경은 원복.**
3. **Stage-2: XGBoost → 간단한 MLP** (교수님 피드백 — tree는 외삽 불가). 타깃 = `margin = fair − MC − recent_margin`. (참고: 실측상 tree가 잔차에 더 나을 수 있으나, 피드백 지시대로 MLP를 기본으로. tree는 비교용으로만 남겨도 됨.)
4. **손실함수 벤치**: DeepONet을 **MSE / MAE / MAPE** 세 손실로 학습·비교(팀: MAE·MAPE가 소폭 우수). 목적별 cost function 재검토.
5. **train/valid = random shuffle** (시간순 X). test는 미래 유지. (검증 자체의 미래참조는 무방 — 국면 과적합 방지.)

## 2. 지표 (metrics) — %error 중심
- **MAPE, bp 오차, MAE, RMSE**, per-stage R²(Stage-1/2/Final), Spearman.
- **moneyness(ATM) 구간별 오차**: 가격(또는 moneyness) 세그먼트별 MAPE·bias.
- **calibration**: pred vs actual 회귀 slope(이상=1)·절편, 저가/고가 skew.
- **이론가(Stage-1)만의 R²·%error를 별도 강조.**

## 3. 속도 실험 (팀 궤적)
- 전체 test set(OOS)을 **MC로 pricing한 총시간** vs **DeepONet 추론 총시간** 비교. (측정조건 명시: CPU 1코어, 상품 간 직렬. MC는 소표본 상품당시간 추정→전체 환산 가능, DeepONet은 전량 실측.)
- 팀 참고치: MC ~5.46s/상품, DeepONet ~4.26µs/상품 → ~130만배. price agreement(corr·MAE) 그림도.

## 4. 산출 그림 (팀원과 동일 구성)
1. **속도**: MC vs DeepONet (로그축 막대) + price agreement scatter + price diff 히스토그램.
2. **fair − MC 히스토그램** (평균 ≈ −512 KRW; MC>FAIR — 원인 IV 미사용).
3. **per-stage R²** (Stage-1 / Stage-2 / Final, 손실 변형별).
4. **저가 케이스 분석** ← 교수님 최신 요청:
   - **공정가 세그먼트별 오차**(mean fair−MC, MAPE) — "저가일수록 오차↑" 정량화(팀: 최저 세그 MAPE ~12.5% → 최고 ~1.9%).
   - **패턴/드라이버**: |오차| vs (coupon, sig_eff, σ₃, b_over_k, tenor, cpn_spread 등) 상관 + 극단 케이스 표.
   - (참고) 캘리브레이션 아이디어: train 역사적 변동성을 세그먼트별 k배 확대 → **단, 회사 데이터에 IV 없어 보류**, 향후 IV 확보 시 반영. 지금은 "분석·패턴 제시"까지만.

## 5. 재실행 순서 (MC 제외)
```
# product_mc.parquet 는 그대로 사용
python -m els_hedging_v1.datasets        # 필요시 재생성(기존 MC 사용)
python -m els_hedging_v1.deeponet --all-losses   # Stage-1: MSE/MAE/MAPE (버그 가드 적용)
python -m els_hedging_v1.margin_mlp      # Stage-2: MLP (신규)
python -m els_hedging_v1.metrics         # %error·moneyness·calibration·per-stage
python -m els_hedging_v1.speed_benchmark # MC vs DeepONet
python -m els_hedging_v1.case_analysis   # 저가 세그먼트·드라이버
python -m els_hedging_v1.figures         # 위 4개 그림
```
> 환경: `KMP_DUPLICATE_LIB_OK=TRUE`, `PYTHONIOENCODING=utf-8`. MC 재실행 없음.

## 6. 성공 기준 (팀 궤적)
- Stage-1 DeepONet MC 재현 R² **~0.95**(현재 0.82 → 튜닝으로 상향), MAE/MAPE 손실이 MSE 이상.
- 속도 그림 + fair−MC(−512) + 저가 오차 패턴 재현.
- 이론가 중심 서사로 정리(Final은 부차로만).
