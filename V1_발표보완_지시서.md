# ELS_hedging_V1 — 발표 보완 지시서 (팀원 덱 대비 부족분 채우기)

> 팀원 발표(SBA_ELS_260713)와 대등하게 만들기 위한 5개 보완. **MC 전체 재실행 금지** — 기존 `data/cache/product_mc.parquet`·예측 parquet 재사용. 속도만 배치로 재측정.

## 1. 부호 있는 편향(over/underprice) 그림  ← 교수님 Q2 대응 (팀 슬라이드 16)
- 기존 저가케이스(04)는 MAPE(크기)만. **부호(signed bias)** 그림을 추가:
  - **공정가(fair) 세그먼트(10분위)별 mean (pred − actual)** 막대 — final 모델 기준. 저가구간 **양수(과대평가) → 고가구간 음수(과소평가)** 패턴을 보이게.
  - 참고로 **mean (fair − MC) by segment**(MC 자체의 저가 과대평가)도 같이. (전부 음수·저가일수록 더 음수 = MC>fair 과대평가)
- 제목: "공정가 구간별 편향 — 저가 과대평가 / 고가 과소평가".

## 2. Predicted vs actual FAIR 산점도 (팀 슬라이드 15)
- **fair(실제) vs 예측(final)** 산점도, walk-forward OOS, `y=x` 대각선. 모델별(final_deeponet_hybrid, final_xgb_hybrid, deeponet_direct 등)로 서브플롯. (기존 01의 DeepONet-vs-MC와 별개)
- 세그먼트 색칠 선택. 왼쪽상단=좋음 해석 가능하게.

## 3. 속도 재측정 (배치 forward) — 팀 4.26µs 급으로
- **DeepONet 추론을 배치 forward**로 재측정: 모델 로드 제외, 전체 OOS를 한 번(또는 큰 배치)에 forward → **µs/상품** 산출(현재 1.62ms/건은 per-call 오버헤드 포함이라 과대). 
- MC는 상품당 시간을 소표본(예: 200건)으로 재어 전체 환산(측정조건 명시: CPU 1코어). 
- 속도 그림(01) 갱신 + "배수" 재계산. (배치 후 팀 130만배 급 기대)

## 4. 표·EDA 채우기 (팀 슬라이드 13·22 + 데이터 개요)
- **모델별 성능 종합표**: 전 모델의 R²·MAE·RMSE·Spearman·MAPE → `data/out/model_comparison.csv` + 표 이미지.
- **필터 cascade 표**(팀 슬라이드 22처럼): 단계별 탈락 사유·건수(노낙인 제외, 티커 매핑 실패, 180일 변동성 부족, 스케줄 없음, 가격이력 없음) → 최종 건수. `data/out/filter_cascade.csv` + 표.
- **EDA 그림**: 연도별 발행건수, 구조유형(스텝다운/리자드), 공정가 분포, 만기·쿠폰 분포, 기초자산 top15 → `data/out/figures/eda_*.png`.

## 5. Stage-1 agreement 개선 (0.927 → ~0.98)
- price agreement 산점도(01 가운데 패널)를 **MAE 손실 최적 DeepONet(R²≈0.947)** 기준으로 통일. (현재 기본 손실본이라 0.927) → corr 상승.
- 손실 벤치표(MSE/MAE/MAPE)는 그대로 유지하되, **대표 그림·수치는 MAE 최적본**으로 일관.

## 산출·확인
- 새/갱신 그림은 `data/out/figures/`, 표는 `data/out/*.csv`. 끝나면 갱신된 속도 배수·agreement corr·편향 그림 요약 보고.
- ⚠ MC 전체 재실행 없음. 필요한 건 예측 재사용 + 속도 소표본 재측정뿐.
